# RetroCause GitHub 开源引流策略（公开文档）

> 更新日期：2026-04-01
> 用途：公开说明项目的开源价值、传播路径与社区吸引点。

---

## 1. 开源目标

RetroCause 的 GitHub 开源目标不是单纯“展示代码”，而是：

1. 吸引开发者与早期爱好者试用
2. 通过可运行 demo 传播“Why did this happen?” 的产品心智
3. 为后续消费级产品和高级版能力积累社区与反馈

---

## 2. 开源版本应突出什么

开源版应该重点突出：

- 证据驱动的因果分析流程
- 可运行 Demo
- 因果图谱可视化
- 竞争假说链
- 基础反事实 / 干预分析

避免把开源首页写成纯技术堆栈说明，应该用更容易传播的表达：

> Ask any “Why did this happen?” question and get an evidence-backed causal map.

---

## 3. GitHub 引流的核心抓手

### 3.1 Demo First

必须保证：

- 本地运行简单
- 无 API key 也能看 Demo
- 有 3-5 个高质量示例问题

### 3.2 Show, don’t tell

仓库首页优先展示：

- 输入问题
- 结果截图
- 因果图
- 干预/what-if 结果对比
- 因子影响分析（slider / before-after comparison）

### 3.3 传播关键词

推荐围绕这些表达构建 README / tagline：

- Why did this happen?
- Evidence-backed causal explainer
- Counterfactual and intervention analysis
- Competing explanations, not just one answer

---

## 4. 社区传播路径

建议顺序：

1. GitHub README 与截图完善
2. Demo 页面可公开访问
3. Product Hunt / Hacker News / Reddit / X 分发
4. 发布带示例问题的短内容

---

## 5. 开源版与后续商业化的关系

开源版用于：

- 传播产品心智
- 获得开发者反馈
- 验证用户是否真的对“why questions”有持续兴趣

后续商业版用于：

- 更强的质量与稳定性
- 更深的情景模拟与批量分析
- 更强的模板、资产沉淀与体验闭环

核心原则：

> 开源版负责让人喜欢上产品，商业版负责让高频用户离不开产品。
